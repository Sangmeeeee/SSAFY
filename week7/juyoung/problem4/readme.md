# 1. 알고리즘 설명

**전체 수열의 길이** `len`와 중간에 들어가는 **o가 k+2개인 수열의 길이** `temp`를 이용해서 문제를 풀 수 있다.

우선 탐색할만큼의 수열의 길이를 계산한다. 즉, N을 포함하는 수열의 최소 길이를 구한다.  
이는 문제에서 `S(k) = len(S(k-1)*2)+k+3`라는 식을 도출하여 구할 수 있었다.

Moo수열은 세가지 부분으로 나눠서 볼 수 있다.

1. S(k-1) 수열(=이전 수열 S(k))
2. o가 k+2개인 수열
3. S(k-1) 수열(=이전 수열 S(k))

:white_check_mark: **만약 N이 2번 수열에 속한다면**  
2번 수열은 가장 첫번째 원소만 'm'이고 나머지는 전부 'o'이므로,  
`N-이전 수열의 길이`가 1일 때 'm', 1이 아닐 때 'o'를 출력하고 종료한다.  
이때 이전 수열의 길이 `prev`는 전체 길이에서 o가 k+2개인 수열의 길이를 뺀 값을 2로 나눈 것으로, `(len-temp)/2`이다.

:white_check_mark: **만약 N이 1, 3번 수열에 속한다면**  
N이 2번 수열에 속할 때까지 전체 수열의 길이와 2번 수열의 길이를 조정한다.  
- N이 1번 수열에 속한다면 전체 문자열의 길이 `len`를 이전 문자열의 길이 `prev`로 수정하고, o가 k+2개인 수열의 길이 `temp`를 1만큼 감소시킨다.
- N이 3번 수열에 속할 때도 마찬가지이지만, 이때는 앞쪽의 수열들이 빠지기 때문에 `N`에서도 그만큼의 길이를 빼줘야 한다.
