# 1. 알고리즘 설명

전체적인 풀이 과정은 아래와 같다.

1. **주어진 문자열 `str1`과 `str2`를 두 글자씩 끊어서 집합 `A`와 `B`를 만든다.**
	- `A`와 `B`는 map으로, 두 글자씩 끊은 문자열의 개수를 카운트한다.
	- 이 과정에서 중복을 제외한 두 집합의 전체 문자열을 백터 `total`에 저장한다.
2. **두 집합 모두 공집합이라면 `65536`을 반환한다.**
3. **전체 문자열 `total`을 탐색하면서 교집합, 합집합의 개수를 구한다.**
	- 만약 `total`에서 현재 탐색한 문자열이 `temp`이라고 가정하자.
	- 교집합의 개수는 두 집합이 가지는 `temp`의 개수 중 더 적은 것에 해당한다.
	- 합집합의 개수는 두 집합이 가지는 `temp`의 개수 중 더 큰 것에 해당한다.
4. **정답을 구한다.**
	- `(교집합의 개수 / 합집합의 개수) * 65536`을 반환한다.

교집합 구하는 부분에서 약간 애를 먹었다.  
처음에는 map에 다 때려박고 map의 size를 합집합의 개수,  
map에서 1보다 큰 요소들을 카운트한 개수를 교집합의 개수로 생각하고 계산했는데  
이렇게 하면 원소가 중복되는 경우 다른 답이 나왔다.  
그래서 교집합, 합집합 구하는 부분만 다른 분 코드를 참고했다...(눈물)
