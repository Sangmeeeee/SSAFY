# 1. 알고리즘 설명

재귀로 풀다가 잘 안돼서 반복문으로 풀었다.

### 사용 변수
- `map` : 장소의 상태
- `visited` : 청소(방문) 여부
- `isCleaned` : 현재 위치에서 네 방향 중 청소된 곳이 있는지 체크
- `y`, `x`, `dir` : 로봇 청소기의 현재 위치, 방향
- `nx`, `ny`, `nd` : 로봇 청소기가 이동할 위치, 바라볼 방향
- `answer` : 로봇 청소기가 청소하는 칸의 개수

### 전체적인 풀이 방법
1. 아직 현재 위치가 청소되지 않았다면 `answer`를 1만큼 증가시키고 현재 위치의 청소 여부를 갱신한다.
2. 현재 방향 `dir`부터 시작해서 네 방향 `nd`를 돌면서 청소할 공간이 있는지 탐색한다.
3. 청소할 공간이 있으면 현재 위치와 방향을 청소할 공간으로, `isCleaned`를 `true`로 갱신하고 탐색을 종료한다.
4. 현재 위치에서의 네 방향 모두 청소되었다면, (`isCleaned`가 `false`라면)
	1. 현재 바라보는 방향의 반대에 벽이 있는지 확인한다.
	2. 벽이 있다면 후진할 수 없으므로 그대로 중단한다.
	3. 벽이 없다면 방향은 유지하고 현재 위치 `y`, `x`만 한 칸 뒤로 갱신한다.
5. 중단될 때까지 1~4를 반복한다.
