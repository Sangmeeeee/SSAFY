# 1. 알고리즘 설명

플로이드 와샬 알고리즘과 조합을 이용해서 풀었다.

:rose:  
우선 입력을 통해 **인접 행렬**을 만든다.  
입력이 끝난 후 자기 자신이 아니면서 인접 행렬의 원소가 0인 곳,  
즉 연결되지 않은 부분은 충분히 큰 수로 초기화한다.

:rose:  
**플로이드 와살** 알고리즘을 통해 건물들 간의 최소 이동 시간으로 인접 행렬을 갱신한다.

:rose:  
N개 중에 2개를 고르는 **모든 조합에 대해**  
고른 2개의 **치킨 집과 각 건물 사이의 최소 왕복 시간을 구한다.**

`minTime` 배열에 최소 왕복 시간을 저장하는데,  
현재 인덱스가 치킨집으로 설정한 곳이면 0을 저장하고  
그렇지 않다면, 인접 행렬에서 첫번째 치킨집과 두번째 치킨집 중 더 가까운 곳의 이동 시간을 저장한다.  
(인접 행렬에는 왕복이 아닌 편도 이동 시간이 저장되어 있기 때문에 *2를 해주어야 한다.)

이후 `minTime` 배열의 모든 원소의 합 `timeSum`을 구하고  
`timeSum`이 현재 최소 왕복 시간인 `answer`보다 작다면,  
현재 최소 왕복 시간 `answer`와 치킨집의 건물 번호를 갱신한다.
