# 1. 알고리즘 설명

head와 number, 파일 이름을 저장하는 벡터를 만들고 직접 정의한 `compare()` 함수로 정렬하여 풀었다.

tail은 정렬하는 데 필요없기 때문에 head와 number를 저장하고,  
answer 벡터에 답을 삽입하기 위해 filename까지 총 세가지를 저장하는 벡터 `split`를 선언했다.  

파일을 하나씩 탐색하며 파일 이름에서 head와 number에 해당하는 부분을 추출하여  `split`에 삽입한다.  
number에서 0의 처리는 그냥 `stoi()` 함수를 사용했다.  

`compare()` 함수는 첫번째 요소(head)가 같으면 두번째 요소(number)로 비교한 값을 반환,  
그렇지 않으면 첫번째 요소로 비교한 값을 반환하도록 작성했다.  

그래서 `split`에 모든 파일에 대한 값들을 넣고 난 후 `compare()` 함수로 정렬하고,  
정렬한 값에서 세번째 요소인 파일 이름을 차례로 `answer`에 삽입했다.  

:star: **틀린 이유**  
처음에 테케 1, 2번 빼고는 다 틀렸다고 뜨길래 당황했는데  
알고보니 tail을 정렬 기준으로 삼지 않아도 `sort()` 함수는 불안정 정렬이라 정렬 도중 순서가 바뀌는 경우가 있었다.  
그래서 불안정 정렬인 `stable_sort()`를 사용했더니 통과했다.
