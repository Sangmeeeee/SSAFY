# 1. 알고리즘 설명

1. `n`을 `k`진수로 바꾼다.
	1. `n`이 0보다 큰 동안 아래 과정을 반복한다.
		- 문자열 `number`에 `n%k` 값을 덧붙인다.
		- `n`에 `n/k`를 저장한다.
	2. `number`를 뒤집는다.
2. `number`에서 10진수 소수를 찾는다.
	- `number[i]`가 0이 아니면 `temp`에 `number[i]`를 덧붙인다.
	- `number[i]`가 0이면 `temp`가 빈 문자열이 아니고 `temp`가 소수인 경우 `answer`를 1 증가시킨다.
	- `temp`를 빈 문자열로 초기화시킨다.
3. 마지막으로  `temp`가 빈 문자열이 아니라면 남은 문자열에 대해 한번 더 소수 여부를 판별한다. 
	- 소수라면 `answer`를 1만큼 증가시킨다.

`k`진수로 바꾸고 난 후 소수를 판별할 때 인자로 넘어가는 수가 int의 범위를 벗어날 수 있었는데,  
이 부분을 생각 못해서 계속 1번, 11번 테스트케이스에서 실패가 떴다.

따라서 `isPrime` 함수의 인자를 long long 타입으로 바꾸고,  
`isPrime`을 호출할 때 `temp`를 정수로 형변환 시키는 것도 `stoi`가 아닌 `stoll`로 바꿨다.
