# 1. 알고리즘 설명

문제에서 주어진 순서대로 따라 작성하면 쉽게 풀리는 문제였다.  

:rose:  
이 문제에서는 `v`라는 문자열을 가지고 함수를 재귀적으로 호출한다.  
따라서 편의를 위해 `solution()` 이외에 문자열 `v`를 인자로 받는 `solve()` 함수를 선언하여 사용했다.  
그리고 `solution()` 함수에서 주어진 입력 `p`를 `solve()`의 인자로 넘겼다.

:rose:  
2번에서 문자열을 두 "균형잡힌 괄호 문자열" `u`, `v`로 분리할 때,  
`u`에 더 이상 분리할 수 없는 균형잡힌 괄호 문자열을 넣기 위해서 정수형 변수 `left`와 `right`를 사용했다.  

문자열 `v`를 끝까지 탐색하는데, 이때 열린 괄호('(')가 나오면 `left`를 1만큼 증가,  
닫힌 괄호(')')가 나오면 `right`를 1만큼 증가시켰다.  
이와 동시에 `u`라는 문자열을 새로 선언하여 각각 맞는 괄호를 삽입해주었다.  

`left`와 `right`가 같아지는 순간이 균형잡힌 괄호 문자열이 되는 순간이므로, 이때 break 한다.

`v`에는 기존 문자열이 모두 저장된 상태이기 때문에 `u`에 포함되는 부분을 지운다.  
`u`에는 `left`+`right` 길이만큼이 저장되었으므로 `v`에서 앞쪽을 그만큼 지워준다.

:rose:  
3번에서 문자열 `u`가 올바른 괄호 문자열인지 검사할 때는 정수형 변수 `flag`를 사용했다.  
0으로 초기화된 상태에서, `u`를 탐색하며 열린 괄호일 때는 `flag++`, 닫힌 괄호일 때는 `flag--`를 한다.  
만약 `flag`가 0보다 작아지면 올바른 괄호 문자열이 아니므로 곧바로 break한다.

이후 과정에서 `flag`가 0인지 아닌지의 여부에 따라 처리하는 게 달라진다.

:rose:  
위 부분을 제외한 나머지는 정말 문제에 적힌 그대로 구현하면 되는 부분이었다.
