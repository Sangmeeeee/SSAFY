# 1. 알고리즘 설명

`stringstream`을 이용하여 각 행동에 맞는 개수만큼을 입력받았다. 

map `users`에 각 아이디별 닉네임을 저장한다.  
벡터 `uid`에는 각 행동을 한 아이디를 차례로, 벡터 `answer`에는 각 행동에 대한 문장을 차례로 삽입한다.  
마지막에 벡터를 순회하며 `uid`의 각 아이디에 맞는 닉네임을 map에서 찾고,  
알맞은 닉네임 + `ansewr`에 저장된 문장을 출력한다.

1. `Enter`인 경우
	- 아이디와 닉네임을 입력받는다.
	- map에 아이디를 key로 하고 닉네임을 value로 하는 값을 삽입한다.
	- `uid` 벡터에 아이디를 삽입한다.
	- `answer` 벡터에 `"님이 들어왔습니다."`를 삽입한다.
2. `Leave`인 경우
	- 아이디만 입력받는다.
	- `uid` 벡터에 아이디를 삽입한다.
	- `answer` 벡터에 `"님이 나갔습니다."`를 삽입한다.
3. `Change`인 경우
	- 아이디와 닉네임을 입력받는다.
	- map에서 아이디에 해당하는 값을 새로 입력받은 닉네임으로 갱신한다.
